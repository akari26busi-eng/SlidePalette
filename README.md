# 🎨 SlidePalette - スライドの配色提案システム

## 📝 概要
**SlidePalette**は，スライドの原稿や発表内容のテキストを入力するだけで，その内容のイメージに合致した「スライドの配色（カラーパレット）」を自動提案するWebアプリケーションです．
見ている人が内容を直感的にイメージしやすくなる配色を提案し，スライド作成における「色を決める時間」を大幅に短縮することを目指しました．

## 💡 開発のきっかけ・目的
* スライドを作成する際，配色に悩む時間を削減したい．
* 例えば，魚についてのスライドでは，赤よりも青を用いた方がイメージに合っていてわかりやすい．
* 内容にマッチした配色は，聴衆の理解を助け，プレゼンテーションの効果を高めることができると考えたため．

## ⚙️ システム設計・処理フロー
ユーザーが入力したテキストから、以下のステップで色を推定・提案します。

1. **テキスト入力 (Streamlit)**: ユーザーがWebUIからスライドの内容を入力。
2. **重要語抽出 (MeCab / TF-IDF)**: 入力された文章を形態素解析し、TF-IDFを用いて文章内の重要な単語（名詞・動詞・形容詞）を抽出。
3. **類似単語の取得 (Word2Vec)**: 抽出した重要語に対し、Word2Vecの学習済みモデルを用いて意味の近い類似単語を取得。
4. **色推定**: 事前に定義した色と単語の紐付けデータ（`color_data.csv`）と類似単語を照らし合わせ、最適なRGB値/カラーコードを提案。

## 🛠 使用技術
* **言語**: Python 3.x
* **フロントエンド・Webフレームワーク**: Streamlit
* **自然言語処理**: 
  * MeCab (形態素解析)
  * scikit-learn (TF-IDFによる重要語抽出)
  * Gensim / Word2Vec (単語のベクトル化・類似度計算)
* **データ処理**: Pandas

## ✨ 工夫した点
* **単語のベクトル化による柔軟な色推定**: 入力された単語が色データ（`color_data.csv`）に直接存在しなくても、Word2Vecによる類似度計算を挟むことで、文脈や意味合いが近い色を柔軟に引き出せるように設計しました。
* **不要語（ストップワード）の徹底的な除外**: TF-IDFを用いた分析の精度を上げるため、正規表現を用いた記号の削除や、汎用的な名詞・動詞をストップワードとして細かく定義し、本当に意味のある単語だけを抽出できるよう前処理にこだわりました。

## 🌱 今後の課題と展望
* 単語によっては一般的なイメージと異なる色が出力されるケースがある（例：「ペンギンの飼育方法」に対する出力結果の違和感など）。色データの拡充や、ベクトル空間における単語と色の距離計算アルゴリズムの調整が必要です。
* 複数色が提案された際の「配色バランス（ベースカラー、メインカラー、アクセントカラー）」までの提案機能の拡張。
